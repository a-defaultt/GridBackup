# Project Specification: Enterprise-Grade Native Backup Agent (Restic-Powered)

This document outlines the complete architectural design, code specifications, script logic, and deployment workflows for an automated, production-ready, host-level backup appliance.

This plan is written as an explicit implementation guide for AI code-generation engines (such as Antigravity CLI, Aider, or Claude Engineer).

---

## 1. Project Architecture Overview

To achieve absolute filesystem consistency, complete security separation, and robust error management across heterogeneous production servers, this appliance runs **natively on the host OS** rather than within a container.

```
+------------------------------------------------------------------------+
|                            HOST OPERATING SYSTEM                       |
|                                                                        |
|   +-----------------------+                    +-------------------+   |
|   |   Systemd Engine      |                    |  Docker Engine    |   |
|   |  (Timers & Services)  |                    | (Client Stacks)   |   |
|   +-----------+-----------+                    +---------+---------+   |
|               | (Triggers)                               |             |
|               v                                          | (Queries)   |
|   +-----------+-----------+                              |             |
|   |     resticprofile     +------------------------------+             |
|   |   Execution Engine    |                                            |
|   +-----+-----------+-----+                                            |
|         |           |                                                  |
| (Hooks) |           | (Direct File Reads)                              |
|         v           v                                                  |
|  +------+--+    +---+---------------------+                            |
|  | Bash    |    | /var/lib/docker/volumes |                            |
|  | Hooks   |    | /etc, /opt, etc.        |                            |
|  +---------+    +-------------------------+                            |
|                                                                        |
+-----------------------------------+------------------------------------+
                                    |
                                    | (Secure Uploads via TLS)
                                    v
                 +------------------+------------------+
                 |            REMOTE STORAGE           |
                 |  (S3 / Wasabi Object-Locked Repo)   |
                 +-------------------------------------+

```

### Key Security & Resiliency Vectors

* **Zero Container Escape Risk:** By running on the bare-metal host, the agent interacts with the Docker CLI natively without exposing `/var/run/docker.sock` to an arbitrary containerized network stack.
* **Filesystem Fidelity:** File ownerships ($UID$/$GID$) and ACLs are completely preserved during backup and accurately restored, bypassing container mount virtualization anomalies.
* **Resource Defensibility:** Every job executes inside targeted Linux cgroups under a native systemd unit file, protecting the host system from RAM exhaustion or CPU starvation.

### C. `config/profiles.yaml`

Designed utilizing the `resticprofile` structural specifications. It features standard logging outputs and execution inheritance schemas.

```yaml
# yaml-language-server: $schema=https://creativeprojects.github.io/resticprofile/jsonschema/config-2.json
version: "2"

global:
  ionice: true
  ionice-class: 2
  ionice-level: 6
  nice: 10
  restic-arguments: "--compression auto"

profiles:
  # Universal Shared Base Parameter Configuration
  base:
    password-env: "RESTIC_PASSWORD"
    env:
      AWS_ACCESS_KEY_ID: "${AWS_ACCESS_KEY_ID}"
      AWS_SECRET_ACCESS_KEY: "${AWS_SECRET_ACCESS_KEY}"
    
    # Scripts executed under the lifecycle hooks framework
    run-before: "bash /etc/resticprofile/scripts/pre-backup.sh"
    run-after: "bash /etc/resticprofile/scripts/post-backup.sh success"
    run-after-fail: "bash /etc/resticprofile/scripts/post-backup.sh failure"

    backup:
      source:
        - "/var/lib/docker/volumes"
        - "/opt"
        - "/etc"
        - "${TEMPORARY_DUMP_DIR}"
      exclude-file:
        - "/etc/resticprofile/excludes"
      exclude-caches: true
      one-file-system: true
      tag:
        - "automated-system-backup"

  # Fast Local Storage Architecture
  local:
    inherit: "base"
    repository: "local:${LOCAL_REPO_PATH}"
    retention:
      before-backup: false
      after-backup: true
      keep-daily: 14
      keep-weekly: 4
      keep-monthly: 12
      prune: true
    check:
      read-data-subset: "5%"

  # Ransomware-Proof Remote S3 Immutable Vault Architecture
  cloud:
    inherit: "base"
    repository: "${REMOTE_S3_REPO}"
    retention:
      before-backup: false
      after-backup: true
      keep-daily: 7
      keep-weekly: 4
      keep-monthly: 2
      prune: true

```

### D. `config/scripts/pre-backup.sh`

This script executes pre-flight checks, runs defensive database autodiscovery, applies runtime resource constraints, and alerts on anomalies.

```bash
#!/usr/bin/env bash
set -euo pipefail

# Import Environment Directives
if [ -f /etc/resticprofile/.env ]; then
    export $(grep -v '^#' /etc/resticprofile/.env | xargs)
fi

echo "[$(date +'%Y-%m-%d %H:%M:%S')] Starting pre-backup procedures..."

# 1. Host Infrastructure Safety Verification
FREE_SPACE_PCT=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')
if [ "$FREE_SPACE_PCT" -gt "$((100 - MIN_FREE_DISK_PERCENT))" ]; then
    echo "CRITICAL ERROR: Host storage space has fallen below critical threshold (${MIN_FREE_DISK_PERCENT}%). Aborting backup sequence." >&2
    exit 1
fi

# Reinitialize temporary work area safely
rm -rf "${TEMPORARY_DUMP_DIR}"
mkdir -p "${TEMPORARY_DUMP_DIR}"
chmod 700 "${TEMPORARY_DUMP_DIR}"

# 2. Complete Database Container Autodiscovery Grid
ALL_CONTAINERS=$(docker ps --format "{{.Names}}")
MUTED_WARNINGS=0

for CONTAINER in $ALL_CONTAINERS; do
    IMAGE_NAME=$(docker inspect --format='{{.Config.Image}}' "$CONTAINER")
    BACKUP_LABEL=$(docker inspect --format='{{index .Config.Labels "backup.db"}}' "$CONTAINER")

    # Match database runtimes signatures
    if [[ "$IMAGE_NAME" =~ (postgres|postgres:|mysql|mysql:|mariadb|mariadb:) ]]; then
        
        # Guard Clause: Verify Container explicit declaration
        if [ -z "$BACKUP_LABEL" ] || [ "$BACKUP_LABEL" == "<no value>" ]; then
            echo "WARNING: Untagged database engine discovered running in container [$CONTAINER]. Backup state unknown!" >&2
            curl -H "Title: UNPROTECTED DATABASE FOUND" \
                 -H "Priority: high" \
                 -H "Tags: warning,database" \
                 -d "Container [$CONTAINER] matches database signature but lacks 'backup.db' label." \
                 "${NOTIFY_WEBHOOK_URL}" || true
            MUTED_WARNINGS=$((MUTED_WARNINGS + 1))
            continue
        fi

        # 3. Parameterized Safe Isolation Parsing Engines
        case "$BACKUP_LABEL" in
            "postgres")
                echo "Executing transactional atomic extraction for Postgres target [$CONTAINER]..."
                DB_USER=$(docker inspect --format='{{range .Config.Env}}{{if RoyalEnv "POSTGRES_USER"}}{{.}}{{end}}{{end}}' "$CONTAINER" | cut -d= -f2)
                DB_USER=${DB_USER:-postgres}
                
                if ! timeout "${MAX_DB_DUMP_DURATION_MINUTES}m" docker exec "$CONTAINER" pg_dumpall -U "$DB_USER" > "${TEMPORARY_DUMP_DIR}/${CONTAINER}_postgres.sql"; then
                    echo "ERROR: PostgreSQL logical dump failed or timed out on container [$CONTAINER]" >&2
                    exit 2
                fi
                ;;

            "mysql"|"mariadb")
                echo "Executing locks-deferred application dump for MySQL/MariaDB target [$CONTAINER]..."
                if ! timeout "${MAX_DB_DUMP_DURATION_MINUTES}m" docker exec "$CONTAINER" mysqldump --all-databases --single-transaction --quick -u root -p"$(docker inspect --format='{{range .Config.Env}}{{if json . | grep MYSQL_ROOT_PASSWORD}}{{print .}}{{end}}{{end}}' "$CONTAINER" | cut -d= -f2)" > "${TEMPORARY_DUMP_DIR}/${CONTAINER}_mysql.sql" 2>/dev/null; then
                    # Fallback variant syntax approach 
                    if ! timeout "${MAX_DB_DUMP_DURATION_MINUTES}m" docker exec "$CONTAINER" sh -c 'exec mysqldump --all-databases --single-transaction --quick -uroot -p"$MYSQL_ROOT_PASSWORD"' > "${TEMPORARY_DUMP_DIR}/${CONTAINER}_mysql.sql"; then
                        echo "ERROR: MySQL/MariaDB database dump failed or timed out on container [$CONTAINER]" >&2
                        exit 3
                    fi
                fi
                ;;
        esac
    fi
done

echo "[$(date +'%Y-%m-%d %H:%M:%S')] Pre-backup processing successfully finished."

```

### E. `config/scripts/post-backup.sh`

Handles reporting, telemetry routing, cleanup protocols, and third-party monitoring verification.

```bash
#!/usr/bin/env bash
set -euo pipefail

if [ -f /etc/resticprofile/.env ]; then
    export $(grep -v '^#' /etc/resticprofile/.env | xargs)
fi

STATUS="${1:-failure}"

if [ "$STATUS" == "success" ]; then
    echo "[$(date +'%Y-%m-%d %H:%M:%S')] Backup executed successfully. Dispatching telemetry pings."
    
    # Notify Healthchecks.io
    curl -fsS --retry 3 "${HEALTHCHECKS_IO_URL}" || true
    
    # Notify ntfy channel
    curl -H "Title: Backup Successful - ${CLIENT_NAME}" \
         -H "Priority: default" \
         -H "Tags: white_check_mark,automated" \
         -d "Host [${HOST_IDENTIFIER}] completed backup profiles to local and cloud matrices smoothly." \
         "${NOTIFY_WEBHOOK_URL}" || true
else
    echo "[$(date +'%Y-%m-%d %H:%M:%S')] CRITICAL ERROR: Backup pipe reported an operational failure." >&2
    
    # Signal failure tracking states
    curl -fsS --retry 3 "${HEALTHCHECKS_IO_URL}/fail" || true
    
    curl -H "Title: BACKUP CRITICAL FAILURE - ${CLIENT_NAME}" \
         -H "Priority: max" \
         -H "Tags: x,skull,fire" \
         -d "Host [${HOST_IDENTIFIER}] failed execution loop. Review system metrics immediately with: journalctl -u resticprofile-backup" \
         "${NOTIFY_WEBHOOK_URL}" || true
fi

# Wipe transient state databases safely
rm -rf "${TEMPORARY_DUMP_DIR:?}"/*

```

## 4. Idempotent Core System Installer (`install.sh`)

This shell script manages software requirements, sets secure folder permissions, links configurations into `/etc/`, and provisions native Systemd execution timelines.

```bash
#!/usr/bin/env bash
set -euo pipefail

# Ensure execution as privileged root
if [ "$EUID" -ne 0 ]; then
    echo "Deployment configuration required execution via sudo / root access rules." >&2
    exit 1

```

```bash
fi

echo "=== Commencing Installation of Client Backup Agent ==="

# 1. Dependency Acquisition Grid
DEPENDENCIES=(curl jq docker sed grep)
for CMD in "${DEPENDENCIES[@]}"; do
    if ! command -v "$CMD" &> /dev/null; then
        echo "Missing essential system primitive: $CMD. Please deploy via package manager before continuing." >&2
        exit 1
    fi
done

# 2. Dynamic Component Binary Deployment
RESTIC_CURRENT_VERSION="0.17.0"
RESTICPROFILE_CURRENT_VERSION="0.31.0"

if ! command -v restic &> /dev/null; then
    echo "Acquiring Restic Engine Core Binary..."
    curl -L -s "https://github.com/restic/restic/releases/download/v${RESTIC_CURRENT_VERSION}/restic_${RESTIC_CURRENT_VERSION}_linux_amd64.bz2" | bunzip2 > /usr/local/bin/restic
    chmod +x /usr/local/bin/restic
fi

if ! command -v resticprofile &> /dev/null; then
    echo "Acquiring Resticprofile Orchestrator Binary..."
    curl -L -s "https://github.com/creativeprojects/resticprofile/releases/download/v${RESTICPROFILE_CURRENT_VERSION}/resticprofile_${RESTICPROFILE_CURRENT_VERSION}_linux_amd64.tar.gz" | tar -xz -C /usr/local/bin/ resticprofile
    chmod +x /usr/local/bin/resticprofile
fi

# 3. Folder Infrastructure and Directory Provisioning Matrix
mkdir -p /etc/resticprofile/scripts
mkdir -p /var/tmp/backup_stage
chmod 700 /var/tmp/backup_stage

# Copy Configuration Modules securely
cp config/profiles.yaml /etc/resticprofile/profiles.yaml
cp config/excludes /etc/resticprofile/excludes
cp config/scripts/*.sh /etc/resticprofile/scripts/
chmod +x /etc/resticprofile/scripts/*.sh

# Inject Configuration Variable Profiles safely
if [ ! -f /etc/resticprofile/.env ]; then
    if [ -f .env ]; then
        cp .env /etc/resticprofile/.env
    else
        cp .env.template /etc/resticprofile/.env
        echo "ALERT: Default configurations template moved to /etc/resticprofile/.env. Please tailor secret variables before running."
    fi
    chmod 600 /etc/resticprofile/.env
fi

# 4. Native Linux Systemd Service Generation Blueprint
echo "Generating Native Systemd Unit Profiles..."

cat << 'EOF' > /etc/systemd/system/resticprofile-backup@.service
[Unit]
Description=Resticprofile Automation Agent (Profile: %I)
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
EnvironmentFile=-/etc/resticprofile/.env
ExecStart=/usr/local/bin/resticprofile --config /etc/resticprofile/profiles.yaml --name %i backup
StandardOutput=journal
StandardError=journal
User=root
EOF

[Unit]
Description=Automated Resticprofile Schedule Matrix (Profile: %I)

Persistent=true

[Install]
WantedBy=timers.target

systemctl daemon-reload

# 5. Service Activation Engine Loop
# Instantiates discrete target timers mapped cleanly back over profiles configurations
systemctl stop "resticprofile-backup@local.timer" || true
systemctl stop "resticprofile-backup@cloud.timer" || true

sed -i 's/OnCalendar=%i/OnCalendar=*-*-* 02:00:00/' /etc/systemd/system/resticprofile-backup@local.timer || true
sed -i 's/OnCalendar=%i/OnCalendar=*-*-* 04:00:00/' /etc/systemd/system/resticprofile-backup@cloud.timer || true

systemctl daemon-reload

systemctl enable --now resticprofile-backup@local.timer
systemctl enable --now resticprofile-backup@cloud.timer

echo "=== System Installation Matrix Completed Successfully ==="
```

## 5. Verification Checklist & Test Vectors

### Initial Initialization Execution

To prepare the storage backends, run these manual initialization commands immediately after installation:

```bash
# Initialize local backup repository location
resticprofile --config /etc/resticprofile/profiles.yaml --name local init

# Initialize remote S3 storage repository target 
resticprofile --config /etc/resticprofile/profiles.yaml --name cloud init

```

### Operational Check Routines

To audit execution pipelines or confirm backup integrity, execute these system tests:

```bash
# 1. Manually trigger the local backup profile execution loop
systemctl start resticprofile-backup@local.service

# 2. View running logs for active processes
journalctl -u resticprofile-backup@local.service -f

# 3. Check active Systemd timers to verify next execution window
systemctl list-timers --all | grep resticprofile

# 4. View structural data contents of active snapshots
resticprofile --config /etc/resticprofile/profiles.yaml --name local snapshots

```

### Dispatched Shell Verification Drill

To guarantee database logic recovery states without applying full destructive transformations to real file blocks, use the direct file pipeline test:

```bash
# Verify the backup repository files can mount locally for easy browsing
mkdir -p /mnt/restic-recovery-test
resticprofile --config /etc/resticprofile/profiles.yaml --name local mount /mnt/restic-recovery-test
# Inspect via standard shell command access
ls -la /mnt/restic-recovery-test/
umount /mnt/restic-recovery-test
```
